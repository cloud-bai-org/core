# 任務清單：offering-arrangement

## 資料模型與後端基礎

- [ ] 1. 建立供品分類（OfferingCategory）資料模型與 seed 資料（水果、三牲、糕點、飲品、鮮花等）
- [ ] 2. 建立供品主檔（Offering）資料模型，包含名稱、分類關聯、圖片 URL、描述欄位
- [ ] 3. 建立儀式供品關聯（CeremonyOffering）資料模型，包含儀式 ID、供品 ID、位置座標（x, y 百分比）、類型（虛擬/實體）
- [ ] 4. 建立供品照片（OfferingPhoto）資料模型，包含儀式 ID、照片 URL、拍攝時間
- [ ] 5. 實作供品分類與供品列表的 API 端點（GET）

## 虛擬供品選擇（拜神明模式）

- [ ] 6. 實作供品分類列表 UI 元件，顯示各分類名稱與圖示
- [ ] 7. 實作供品庫瀏覽 UI，支援依分類篩選供品
- [ ] 8. 實作供品選擇功能，將所選供品加入當次儀式供品清單
- [ ] 9. 確保供品選擇功能僅在「拜神明」模式下顯示，「祭拜祖先/親人」模式隱藏

## 虛擬供品拖放擺設（拜神明模式）

- [ ] 10. 實作虛擬供桌 2D 俯視圖佈局元件
- [ ] 11. 實作桌面端拖放功能（HTML5 Drag and Drop API）
- [ ] 12. 實作行動端觸控拖放功能（touch event）
- [ ] 13. 實作供品位置以相對座標（百分比）儲存與還原
- [ ] 14. 實作已擺放供品的位置調整（拖動至新位置）
- [ ] 15. 實作已擺放供品的移除功能（長按/點擊後選擇移除）
- [ ] 16. 實作供品擺設狀態的 API 儲存與讀取

## 實體供品紀錄

- [ ] 17. 實作「拍照記錄」按鈕 UI，觸發系統相機（input capture）
- [ ] 18. 實作照片壓縮與上傳至物件儲存的流程
- [ ] 19. 實作照片與儀式紀錄的關聯儲存
- [ ] 20. 實作 QR Code 掃描功能（整合 html5-qrcode 或同類函式庫）
- [ ] 21. 定義 QR Code 內容格式（供桌識別碼與位置資訊）
- [ ] 22. 實作掃描成功後自動關聯供桌位置至儀式紀錄
- [ ] 23. 實作相機權限被拒時的備援方案（手動上傳照片）
- [ ] 24. 實作 QR Code 掃描失敗時的備援方案（手動輸入供桌編號）

## 計時連動與推播通知

- [ ] 25. 實作焚香計時結束事件與供品紀錄的連動邏輯
- [ ] 26. 實作後端推播通知觸發（FCM / APNs），通知內容：「儀式已完成，請前往供桌領取供品」
- [ ] 27. 實作前端 in-app 通知作為推播未送達的備援

## 測試

- [ ] 28. 撰寫供品分類與選擇功能的單元測試
- [ ] 29. 撰寫拖放擺設功能的互動測試（桌面端與觸控端）
- [ ] 30. 撰寫實體供品拍照與 QR Code 掃描的整合測試
- [ ] 31. 撰寫計時連動推播通知的端對端測試
