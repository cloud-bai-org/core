# 提案：廟宇打卡功能

## 為什麼要做這個變更

雲端拜拜應用目前僅支援線上儀式，缺乏與實體廟宇的連結。許多使用者會實際走訪廟宇參拜，希望能記錄自己的參拜足跡。透過 QR Code 打卡與手動搜尋機制，讓使用者在實體廟宇也能留下數位紀錄，同時結合祈願目的推薦參拜路線，提升參拜體驗的完整性與儀式感。搭配足跡地圖與成就徽章，增加使用者的長期參與動機。

## 變更內容

新增完整的廟宇打卡功能模組，包含：

- **QR Code 掃描打卡**：使用 html5-qrcode 掃描廟宇 QR Code，辨識廟宇 ID 並完成打卡
- **手動搜尋打卡**：提供廟宇名稱搜尋功能，從靜態 JSON 搜尋匹配結果，選擇後完成打卡
- **祈願目的選擇**：打卡成功後可選擇祈願目的（學業、事業、愛情、健康、財運、平安等）
- **參拜路線推薦**：根據祈願目的與廟宇神明產生推薦參拜順序與導覽指引
- **個人足跡地圖**：以地圖呈現已登入使用者曾打卡過的廟宇位置
- **成就徽章系統**：統計參拜歷程，達成特定條件頒發成就徽章並顯示動畫
- **虛構廟宇資料**：所有廟宇名稱使用自訂虛構名稱

## 能力（Capabilities）

- `temple-checkin`

## 影響範圍

- **新增頁面/元件**：QR Code 掃描介面、手動搜尋介面、祈願選擇介面、參拜路線導覽、足跡地圖、成就徽章頁面
- **外部依賴**：html5-qrcode（QR Code 掃描）、地圖元件（足跡地圖顯示）
- **靜態資料**：新增廟宇 JSON 資料檔（虛構廟宇名稱、位置、神明資訊）
- **狀態管理**：新增打卡紀錄狀態、祈願紀錄、成就徽章狀態
- **登入依賴**：足跡地圖與成就徽章功能需已登入使用者（依賴 user-auth）
