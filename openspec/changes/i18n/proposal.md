# 變更提案：i18n

## 為什麼需要這個變更

本應用以台灣民間信仰為核心，但使用者群體不限於中文母語者。日本、韓國及英語系國家對台灣宮廟文化亦有興趣，若僅提供繁體中文介面，將大幅限制應用的可及性。此變更旨在建立完整的多語系基礎架構，支援繁體中文、英文、日文、韓文四種語言，並確保文化內容（神明名稱、供品名稱、節慶說明等）皆有適當的在地化翻譯。

## 變更內容

### 新增功能

1. **四語系支援** - 系統支援繁體中文（zh-TW）、英文（en）、日文（ja）、韓文（ko），預設繁體中文，可自動偵測瀏覽器語言
2. **語言切換功能** - 使用者可在設定中即時切換語言，偏好自動記憶，無需重新載入頁面
3. **UI 文案翻譯管理** - 使用 vue-i18n 管理所有 UI 文案，翻譯檔案按功能模組（namespace）分割
4. **文化內容在地化** - 神明名稱、供品名稱、金紙種類、節慶說明等文化內容皆提供四語翻譯
5. **懶載入語言包** - 非預設語言的翻譯檔案採動態載入，減少初始載入大小

## 能力範圍 (Capabilities)

- `i18n`

## 影響評估

### 影響範圍

- **前端架構**：整合 vue-i18n 與 unplugin-vue-i18n，所有 UI 文案改由翻譯函式（`$t()`）輸出
- **狀態管理**：語言偏好需持久化（localStorage），影響全域狀態
- **路由/頁面**：所有現有頁面的硬編碼文案需替換為 i18n key
- **靜態資料**：神明、供品、金紙、節慶等靜態資料結構需擴充多語欄位
- **建置流程**：需設定 unplugin-vue-i18n 進行編譯時期最佳化

### 相依性

- 假設 project-setup 已完成
- 需安裝 vue-i18n 與 unplugin-vue-i18n
