# 設計文件：燒金紙功能

## 設計背景

本功能為線上拜拜儀式流程的關鍵環節，需要在行動裝置上呈現逼真且流暢的金紙焚燒效果。設計上採用 Canvas 粒子系統搭配 GSAP 動畫控制，提供兩種焚燒互動模式（自動與手動），並透過三檔動畫等級設定來適配不同效能的裝置。

## 設計目標

- 使用 Canvas 2D 建構粒子引擎，渲染火焰、灰燼、煙霧等焚燒特效
- 以 GSAP 作為動畫時間軸控制器，管理金紙投入、燃燒、殆盡的階段轉場
- 自動焚燒模式：GSAP Timeline 依序驅動每張金紙的投入與燃燒動畫
- 手動投入模式：支援觸控拖曳手勢，偵測金紙進入金爐碰撞區域後觸發焚燒
- 不同金紙種類對應不同的粒子參數（顏色、大小、燃燒速度）
- 動畫程度三檔設定：精緻（高粒子數、60fps）、標準（中粒子數、30fps）、簡約（低粒子數、基本動畫）

## 非目標

- 不實作 3D 渲染或 WebGL——僅使用 Canvas 2D 以確保相容性
- 不實作金紙購買或金幣系統——本階段僅處理焚燒互動
- 不支援多人同時焚燒的即時同步功能
- 不處理音效——音效將在獨立變更中處理

## 設計決策

### 決策 1：Canvas 2D 粒子引擎而非 CSS 動畫

**選擇**：使用 Canvas 2D 搭配自建粒子系統

**理由**：
- 焚燒效果需要大量粒子（火花、灰燼、煙霧），CSS 動畫無法高效處理數百個同時運動的粒子
- Canvas 2D 在行動裝置上的相容性優於 WebGL，同時提供足夠的渲染效能
- 自建粒子系統可針對金紙焚燒的視覺需求做精確調校

### 決策 2：GSAP 控制動畫時間軸

**選擇**：以 GSAP Timeline 作為動畫編排引擎

**理由**：
- GSAP 提供精確的時間軸控制，適合編排「投入 -> 點燃 -> 燃燒 -> 殆盡」的多階段動畫序列
- 支援暫停、加速、跳轉等操作，方便實作「自動完成」切換
- 與 Canvas 粒子系統整合時，可透過 GSAP 的 `onUpdate` 回呼驅動粒子參數變化

### 決策 3：觸控拖曳使用指標事件

**選擇**：使用 Pointer Events API 實作手動投入的拖曳手勢

**理由**：
- Pointer Events 統一處理觸控與滑鼠輸入，減少跨裝置的相容性問題
- 搭配 `getBoundingClientRect` 做金爐區域的碰撞偵測，簡單且高效

### 決策 4：動畫程度設定由使用者主動選擇

**選擇**：提供三檔手動選擇，而非自動偵測裝置效能

**理由**：
- 自動偵測裝置效能的準確度不穩定，且可能造成使用者困惑
- 手動選擇讓使用者保有控制權，預設為「標準」檔位

## 風險與緩解

| 風險 | 影響 | 緩解措施 |
|------|------|----------|
| 低階行動裝置上粒子動畫掉幀 | 使用體驗不佳 | 三檔動畫設定 + 簡約模式大幅降低粒子數；監控 FPS 並在低於閾值時自動降級 |
| 手動拖曳手勢與頁面捲動衝突 | 誤觸導致頁面滾動 | 焚燒場景全螢幕呈現並攔截 `touchmove` 預設行為 |
| Canvas 記憶體洩漏 | 長時間使用後記憶體增長 | 粒子物件池回收機制；離開頁面時銷毀 Canvas 上下文 |
| 不同金紙種類的視覺差異不夠明顯 | 使用者感知不到差異 | 每種金紙定義獨立的粒子顏色組、燃燒速度與灰燼形狀參數 |
